Menu Import Format Guide

How to Format Your Menu Files

This guide explains how to properly format your menu data for import into Zentry POS. We support multiple file formats to make it easy to bring your existing menu data into the system.

JSON Format

JSON is the most flexible format and is recommended for full menu imports with all possible fields. Your file should contain an array of menu item objects.

Example:

[
  {
    "name": "Classic Burger",
    "price": 12.99,
    "category": "Burgers",
    "description": "Beef patty with lettuce, tomato, and special sauce",
    "modifiers": ["Add cheese", "Extra patty", "Gluten-free bun"]
  },
  {
    "name": "Caesar Salad",
    "price": 9.99,
    "category": "Salads",
    "description": "Romaine lettuce, croutons, parmesan, and Caesar dressing",
    "modifiers": ["Add chicken", "Add shrimp"]
  },
  {
    "name": "Iced Coffee",
    "price": 3.50,
    "category": "Beverages",
    "description": "Cold-brewed coffee over ice"
  }
]
Pro Tips:
  • Use a JSON validator to ensure your file is formatted correctly
  • At minimum, each item must include "name" and "price"
  • Price should be a number (not a string), e.g. 12.99 (not "12.99")
  • Make sure the entire file is wrapped in square brackets [ ]

CSV Format (Comma-Separated Values)

CSV is ideal for users who manage their menu in spreadsheet applications like Microsoft Excel or Google Sheets. The first row must contain column headers.

Required Column Headers:

  • name: The name of the menu item
  • price: The price of the menu item

Optional Column Headers:

  • category: The category for grouping menu items
  • description: A description of the menu item
  • image: URL or path to an image

Example:

name,price,category,description
Classic Burger,12.99,Burgers,Beef patty with lettuce tomato and special sauce
Caesar Salad,9.99,Salads,Romaine lettuce croutons parmesan and Caesar dressing
Iced Coffee,3.50,Beverages,Cold-brewed coffee over ice
Pro Tips:
  • If any value contains commas, enclose it in double quotes: "Beef patty, lettuce, and tomato"
  • Column headers are case-insensitive (NAME, Name, and name all work)
  • Save your spreadsheet as CSV before importing

Excel Format (XLSX/XLS)

You can import directly from Excel files (.xlsx or .xls). The first row should contain column headers.

Column Headers:

The system looks for these headers (case-insensitive):

For Item Name "Name", "Item", or "Item Name"
For Price "Price", "Cost", or "Amount"
For Category "Category" or "Type"
For Description "Description" or "Details"

Example Format:

Name Price Category Description
Classic Burger 12.99 Burgers Beef patty with lettuce, tomato, and special sauce
Caesar Salad 9.99 Salads Romaine lettuce, croutons, parmesan, and Caesar dressing
Iced Coffee 3.50 Beverages Cold-brewed coffee over ice
Pro Tips:
  • The system will import from the first sheet in your workbook
  • You can include currency symbols in price cells ($12.99)
  • Make sure there are no merged cells in your data range

TXT Format (Text File)

Text files are simple but effective for basic menu imports. Each line represents one menu item with data separated by semicolons (;).

Format:

Item Name;Price;Category;Description

Example:

Classic Burger;12.99;Burgers;Beef patty with lettuce, tomato, and special sauce
Caesar Salad;9.99;Salads;Romaine lettuce, croutons, parmesan, and Caesar dressing
Iced Coffee;3.50;Beverages;Cold-brewed coffee over ice
Pro Tips:
  • Only the first two fields (name and price) are required
  • If you omit the category, the item will be assigned to "Uncategorized"
  • You can include additional fields as needed, but they'll be ignored if not supported

Word Document (DOCX) Format

You can import menu data from a Microsoft Word document. The system will extract the text content from your document and process it.

Format:

Each line in your Word document should follow the same format as the TXT option:

Item Name;Price;Category;Description

Example:

Classic Burger;12.99;Burgers;Beef patty with lettuce, tomato, and special sauce

Caesar Salad;9.99;Salads;Romaine lettuce, croutons, parmesan, and Caesar dressing

Iced Coffee;3.50;Beverages;Cold-brewed coffee over ice

Tips for Word Documents:

  • Use a clean, simple document with one menu item per line
  • Avoid using tables, columns, or special formatting
  • Make sure each line follows the Item;Price;Category;Description format
  • Save your document as a .docx file
Important Notes:
  • The system extracts plain text from Word documents, so any special formatting, images, or complex layouts will be ignored
  • Only the first two fields (name and price) are required
  • For more complex menu structures, consider using the Excel or JSON formats

Menu Data Fields

Here's a comprehensive list of all fields that can be included in your menu imports:

Field Name Description Status
name The display name of the menu item Required
price The price of the menu item (numeric value) Required
category Category for grouping menu items Optional
description Detailed description of the menu item Optional
image URL or file path to an image Optional
modifiers Options or add-ons (JSON only) Optional
available Whether the item is available (true/false) Optional
taxRate Custom tax rate for this item (overrides default) Optional
Important Notes:
  • All formats support the required fields, but some optional fields may only be available in certain formats
  • Array fields like "modifiers" are only fully supported in JSON format
  • Boolean fields should be "true" or "false" (JSON), or "1"/"0" (CSV/TXT)
